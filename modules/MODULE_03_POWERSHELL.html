First Principle: Everything is an Object

🧠 The Core Truth

PowerShell doesn't just spit text like CMD. Everything is an object with properties and methods. Get-Process returns process objects, not strings. You can filter, sort, and manipulate them programmatically.

This is why | Where-Object {$_.CPU -gt 100} works — you're filtering objects by property, not parsing text with regex.

🎯 Layman Translation

CMD is like shouting into a room and reading what echoes back. PowerShell is like asking a librarian for a specific book, then asking that book for its table of contents, then asking for Chapter 3. The librarian (PowerShell) understands structured requests.

From Layman to Low-Level: The Pipeline

# LAYMAN: "Show me the big processes" Get-Process | Sort-Object CPU -Descending | Select-Object -First 5 # LOW-LEVEL: "Enumerate process structures, sort by kernel/user time, extract fields" $procs = Get-Process $sorted = $procs | Sort-Object {$_.CPU} -Descending $top5 = $sorted | Select-Object -First 5 $top5 | Format-Table Name, Id, CPU, WorkingSet # FIRST PRINCIPLE: Objects flow through the pipeline, not text # Each | passes OBJECTS, not strings. This is why PowerShell is a shell for hackers.

Lab Exercise: Build Your Own Recon Pipeline

Build a PowerShell one-liner that finds:

  1. All processes using >100MB RAM
  2. Sort by memory usage descending
  3. Show only Name, Id, WorkingSet64
  4. Export to CSV
Get-Process | Where-Object {$_.WorkingSet64 -gt 100MB} | Sort-Object WorkingSet64 -Descending | Select-Object Name, Id, @{N="RAM_MB";E={[math]::Round($_.WorkingSet64/1MB,2)}} | Export-Csv -Path "memory_hogs.csv" -NoTypeInformation
