Module 02: Linux Basics

MODULE 02/22 ● LIVE TESTED CLASSIFICATION: RESTRICTED

Linux is the operating system of servers, routers, embedded devices, and most of the internet. If you can't navigate Linux, you can't navigate the infrastructure. Windows is the desktop. Linux is the battlefield. This module teaches you to survive on that battlefield.

Imagine you're dropped behind enemy lines. The locals speak a different language, use different tools, and the maps are wrong. That's Linux for a Windows user. But every server, every router, every IoT device runs Linux. Learn it or stay blind.

SECTION 01

The Filesystem

Linux has no C: drive. Everything starts at / — the root. From there, directories branch like a tree. Learn the tree, navigate the system.

// Linux filesystem hierarchy $ tree -L 2 / / ├── bin # Essential user binaries (ls, cat, cp) ├── boot # Boot loader files ├── dev # Device files (disks, terminals) ├── etc # Configuration files │ ├── passwd # User accounts │ ├── shadow # Password hashes (root only) │ ├── hosts # IP to hostname mapping │ └── ssh/ # SSH server config ├── home # User home directories ├── lib # Shared libraries ├── media # Removable media mount point ├── mnt # Temporary mount point ├── opt # Optional software ├── proc # Process information (virtual filesystem) ├── root # Root user home directory ├── run # Runtime variable data ├── sbin # System binaries (fdisk, reboot) ├── srv # Service data ├── sys # System information ├── tmp # Temporary files ├── usr # User programs │ ├── bin # Non-essential binaries │ ├── local # Locally installed software │ └── share # Shared data └── var # Variable data ├── log # Log files ├── mail # User mail └── spool # Spool directories // Key directories for attackers: // /etc/passwd — user list (no passwords) // /etc/shadow — password hashes (root only) // /var/log — evidence of your actions // /tmp — writable by everyone, good for staging // /proc — live process info, no disk footprint
Why this matters

On Windows, you know Program Files, Users, Windows. On Linux, everything is different. But the critical files are in predictable places. /etc/passwd always exists. /var/log always exists. Learn these locations and you can find what you need on any Linux system.

SECTION 02

Essential Commands

These are the commands you'll use every day. No GUI, no clicking. Just type and execute.

// Navigation $ pwd # Print working directory $ cd /etc # Change directory $ ls -la # List all files (including hidden) $ ls -la /tmp # List /tmp directory // File operations $ cat /etc/passwd # Display file contents $ head -20 /var/log/auth.log # First 20 lines $ tail -f /var/log/syslog # Follow log in real-time $ grep "root" /etc/passwd # Search for string $ find / -name "*.conf" 2>/dev/null # Find files by name // Process management $ ps aux # All processes $ ps aux | grep ssh # Filter for ssh $ top # Interactive process viewer $ kill -9 1234 # Force kill process 1234 // Network $ ifconfig # Network interfaces (legacy) $ ip addr # Network interfaces (modern) $ netstat -tulpn # Listening ports with processes $ ss -tulpn # Same, modern replacement $ curl -I http://target.com # HTTP headers // File permissions $ chmod 755 script.sh # rwxr-xr-x $ chown root:root file.txt # Change owner $ ls -la /etc/shadow # Check permissions -rw-r----- 1 root shadow 1234 Jan 15 08:23 /etc/shadow
# Quick reference card — print and tape to your monitor NAVIGATION: pwd — where am I cd ~ — go home cd - — go back ls -la — list everything FILES: cat file — show contents less file — scroll through contents head -n 10 — first 10 lines tail -n 10 — last 10 lines grep "text" — search for text find / -name — find files PROCESSES: ps aux — all processes top — live view kill -9 PID — kill process NETWORK: ip addr — my IP ss -tulpn — listening ports curl url — fetch web page wget url — download file PERMISSIONS: chmod 777 — everyone can do everything (DANGER) chmod 600 — only owner can read/write chown user — change owner
Why pipes matter

The | character (pipe) is Linux's superpower. It takes the output of one command and feeds it into another. ps aux | grep ssh means "show all processes, then filter for ssh." cat /var/log/auth.log | grep "Failed password" | wc -l means "show auth log, find failed passwords, count them." This is how you build complex queries from simple tools.

SECTION 03

Users and Permissions

Linux is multi-user by design. Every file has an owner, a group, and permissions. Understanding this is critical for privilege escalation.

// /etc/passwd — user accounts (world-readable) $ cat /etc/passwd root:x:0:0:root:/root:/bin/bash daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin bin:x:2:2:bin:/bin:/usr/sbin/nologin sys:x:3:3:sys:/dev:/usr/sbin/nologin gwu07:x:1000:1000:George Wu,,,:/home/gwu07:/bin/bash // Format: username:password:UID:GID:GECOS:home:shell // x = password stored in /etc/shadow // UID 0 = root (god mode) // UID 1000+ = normal users // /etc/shadow — password hashes (root only) $ sudo cat /etc/shadow root:$6$rounds=5000$...:18723:0:99999:7::: gwu07:$6$rounds=5000$...:18723:0:99999:7::: // Format: username:hash:lastchange:min:max:warn:inactive:expire // $6$ = SHA-512 hash // $rounds=5000$ = 5000 iterations (slow to crack) // File permissions $ ls -la /etc/passwd -rw-r--r-- 1 root root 2875 Jan 15 08:23 /etc/passwd // - = regular file // rw- = owner can read, write, not execute // r-- = group can read only // r-- = others can read only // SUID — run as file owner, not current user $ find / -perm -4000 2>/dev/null /usr/bin/sudo /usr/bin/passwd /usr/bin/newgrp // 4000 = SUID bit set // When you run passwd, it runs as root (to edit /etc/shadow) // If passwd has a bug, you can exploit it to become root

SECTION 04

Lab Exercise: Linux Survival

Practice these commands on any Linux system — VM, VPS, or WSL on Windows.

  1. Navigate to /etc and list all files starting with "pass"
  2. Read /etc/passwd and identify all users with /bin/bash shell
  3. Find all SUID files on the system
  4. Check what ports are listening with ss -tulpn
  5. Follow /var/log/auth.log in real-time with tail -f
  6. Use grep to find "Failed password" attempts
  7. Count how many failed attempts with wc -l
  8. Create a script that shows: hostname, IP, logged-in users, listening ports
#!/bin/bash # linux_survival.sh — System enumeration script echo "=== SYSTEM INFO ===" echo "Hostname: $(hostname)" echo "IP: $(ip addr show eth0 | grep 'inet ' | awk '{print $2}')" echo "Kernel: $(uname -r)" echo "" echo "=== USERS ===" echo "Users with bash shell:" grep "/bin/bash" /etc/passwd | cut -d: -f1 echo "" echo "=== LISTENING PORTS ===" ss -tulpn | grep LISTEN echo "" echo "=== SUID FILES ===" find / -perm -4000 2>/dev/null | head -10 echo "" echo "=== FAILED LOGINS ===" grep "Failed password" /var/log/auth.log 2>/dev/null | tail -5 echo "" echo "=== DONE ==="
KEY TAKEAWAYS

What You Learned

  • Linux is the infrastructure: Servers, routers, IoT — everything runs Linux

  • Windows has Linux too: OpenSSH, WSL — learn both, use both
