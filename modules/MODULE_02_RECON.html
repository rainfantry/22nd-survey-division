Section 1: Recon Before Everything

Recon is not optional. It's not prep. It IS the operation. Every command you run blind is noise. Every pivot you make without a target map is wasted time — and burned opsec.

SKYWALKER answers the five questions you need before any lateral move:

#QuestionSKYWALKER Section
1What machine am I on?§1 System Identity
2What privileges do I have?§2 User & Privilege Context
3Is AV active? What version?§4 Defender / AV Status
4What's on the network? Any pivots?§6 Network State
5What's the patch level? Which CVEs are open?§7 Patch State

Run this before you do anything else. The answers determine your entire move. Defender on? Use HWBP (Module 11). No admin token? Need UAC bypass (Module 08). Domain-joined? Use Kerberoast (Module 03). WORKGROUP? Pivot machine-to-machine via SMB or SSH.

Section 3: winrecon.ps1 — How It Works

The entire script is PowerShell. No binaries. No executables. Pure LOTL — it runs on any Windows machine with PowerShell 5.1+. Seven sections, each a specific intelligence category.

PS> Get-CimInstance Win32_ComputerSystem
# §1: Hostname, manufacturer, model, RAM

PS> whoami /all; Get-LocalUser
# §2: Token, SID, group memberships, all local accounts

PS> Get-MpComputerStatus
# §4: Every Defender protection flag — your greenlight indicator

PS> Get-NetTCPConnection | Where-Object {$_.State -eq 'Listen'}
# §6: All listening ports with PIDs — find C2, DB, admin panels

PS> Get-HotFix | Sort-Object InstalledOn -Descending
# §7: Full patch list, newest first — find the gap
Every line is a cmdlet you could run manually. winrecon.ps1 just chains them, formats the output, and adds the CRITICAL flags. Understanding the underlying cmdlets is the point — the script is the aggregator, not the magic.
Section 4: The Unknown Account — apacw

Finding apacw in the local admins group is the most interesting result of this scan. This account:

PS> Get-LocalUser apacw | Format-List *
Name: apacw
SID: S-1-5-21-2235627559-836241553-1517004315-1001
Enabled: True
PasswordRequired: True
LastLogon: [check this]

PS> net localgroup Administrators
Administrator, apacw, gwu07 # apacw is a local admin

PS> Get-WinEvent -LogName Security | Where-Object {$_.Id -eq 4624 -and $_.Message -match 'apacw'}
# Check last logon events — was this account used recently?

Unknown local admin accounts are either: (a) pre-planted persistence by a prior attacker, (b) a leftover service account, or (c) a forgotten admin account. Each possibility has different implications for your operation.
