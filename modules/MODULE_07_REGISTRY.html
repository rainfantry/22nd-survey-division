Module 7 of 22 — The Windows database of secrets

First Principle: Registry is the OS's Database

🧠 The Core Truth

The Windows Registry is a hierarchical database that stores configuration for the OS, hardware, software, and users. It's not files — it's a memory-mapped database loaded at boot. Every setting, every preference, every autostart program lives here.

Forensics 101: If it happened on Windows, the registry knows. Deletion ≠ erasure — the hive has transaction logs and backups.

🎯 Layman Translation

The registry is like a building's maintenance log. Every tenant (program) writes what they changed: "I installed a new lock," "I changed the thermostat," "I made a copy of the key." Even if the tenant moves out, the log remains. Forensics is reading that log.

Registry Hives: The Five Kingdoms

HKEY_CURRENT_USER (HKCU) — Current user's settings. Moves with the user profile.
HKEY_LOCAL_MACHINE (HKLM) — System-wide settings. Applies to all users.
HKEY_CLASSES_ROOT (HKCR) — File associations and COM objects.
HKEY_USERS (HKU) — All loaded user profiles.
HKEY_CURRENT_CONFIG (HKCC) — Current hardware profile.

📊 Live Test: System Watcher Rollback Behavior

Hypothesis: System Watcher rolls back unauthorized registry changes.

TEST> Write HKCU\...\Run\SWTest_22DIV = notepad.exe Result: Key written successfully TEST> Wait 15 seconds Result: Key still present — NO ROLLBACK TEST> Update key to point to phantom_rpc.exe (privesc tool) Result: Key updated successfully TEST> Execute phantom_rpc.exe --list Result: Process ran, key still present — NO ROLLBACK TEST> Manual cleanup: Remove key Result: Key removed successfully

Conclusion: System Watcher rollback is detection-coupled, not behavior-coupled. A binary that passes KAV static scan (phantom_rpc.exe: 0 detections) will not trigger rollback even when executed from a Run key. Evade detection = evade rollback.

Lab Exercise: Registry Persistence Scanner

Write a C program that:

  1. Enumerates all Run keys (HKCU and HKLM)
  2. Checks if each value points to an existing file
  3. Flags non-existent paths (orphaned persistence)
  4. Checks file signatures on existing binaries
  5. Outputs: [Hive] [KeyName] [Path] [Status]

Challenge: Handle REG_EXPAND_SZ (environment variables like %APPDATA%). Recurse into RunOnce and RunOnceEx keys. Detect suspicious paths (temp folders, appdata, unusual extensions).

#include #include void scan_run_keys(HKEY hive, const char* hiveName) { HKEY hKey; const char* path = "SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Run"; if (RegOpenKeyExA(hive, path, 0, KEY_READ, &hKey) != ERROR_SUCCESS) return; char valueName[256], data[MAX_PATH]; DWORD valueNameSize, dataSize, type; DWORD index = 0; while (1) { valueNameSize = sizeof(valueName); dataSize = sizeof(data); if (RegEnumValueA(hKey, index++, valueName, &valueNameSize, NULL, &type, (LPBYTE)data, &dataSize) != ERROR_SUCCESS) break; if (type == REG_SZ || type == REG_EXPAND_SZ) { // Expand environment variables char expanded[MAX_PATH]; ExpandEnvironmentStringsA(data, expanded, MAX_PATH); // Check if file exists DWORD attribs = GetFileAttributesA(expanded); const char* status = (attribs != INVALID_FILE_ATTRIBUTES) ? "EXISTS" : "ORPHANED"; printf("[%s] %-30s | %-50s | %s\n", hiveName, valueName, expanded, status); } } RegCloseKey(hKey); } int main() { printf("=== Registry Persistence Scan ===\n\n"); scan_run_keys(HKEY_CURRENT_USER, "HKCU"); scan_run_keys(HKEY_LOCAL_MACHINE, "HKLM"); return 0; }

Key Takeaways

  • Registry is the OS diary: Everything leaves a trace
  • Persistence lives in Run keys: HKCU and HKLM are primary targets
  • System Watcher is detection-coupled: No detection = no rollback
  • Service keys = privilege: ImagePath hijacking is SYSTEM escalation
  • Forensics is reading: The same APIs malware uses, used for defense
