Module 08 — Privesc

You land as a user. You leave as SYSTEM. Every privilege is a door. GeoDefend shows you the doors — as an attacker, you walk through them.
Section 1: The Three Paths to SYSTEM

Privilege escalation on Windows has three reliable paths. You need exactly one. Check which privileges are available — that tells you which path to take.

PrivilegeStateAttack
SeImpersonatePrivilege Enabled PrintSpoofer / GodPotato → SYSTEM token
SeDebugPrivilege Enabled OpenProcess(lsass) → credential dump → reuse
SeLoadDriverPrivilege Enabled BYOVD → load vulnerable driver → kernel code exec
SeTakeOwnershipPrivilege Enabled Take ownership of any file → replace binaries
SeBackupPrivilege Enabled Read any file regardless of ACL → SAM/SYSTEM dump
On your lab machine (.92): all 5 of these are enabled for gwu07. SeImpersonatePrivilege alone gives you SYSTEM with PrintSpoofer. You don't need to chain anything.
Section 2: BlueHammer — Windows Defender as the Escalation Vector

BlueHammer is a class of exploits where Windows Defender's SYSTEM-level execution becomes the privilege escalation vector. Defender runs as SYSTEM. It reads and writes files. If you can race those operations (TOCTOU), Defender writes your payload where you want it.

# BlueHammer mechanism (simplified)
1. Defender scans file X as SYSTEM
2. TOCTOU window: Defender reads X, pauses (user-mode oplock)
3. Attacker swaps X for payload via symlink/rename
4. Defender resumes, writes/moves "X" (now payload) as SYSTEM
5. Payload lands in System32 or executes with SYSTEM privileges

Minimum safe Defender version: 4.18.26050.3011
GeoDefend scan found: 4.18.26040.7 on LAPTOP-R32M8MLI (May 2026)
→ That machine was vulnerable to BlueHammer class
This is the same class of vulnerability as VULN-195458 (MSRC). George Wu discovered it via hardware debug register analysis while building the HWBP bypass. The mechanism is different but the root cause is the same: Defender's SYSTEM execution creates privilege escalation opportunities at the interface between user and kernel mode.
