Module 01: Reconnaissance & OSINT

MODULE 01/22 ● LIVE TESTED CLASSIFICATION: RESTRICTED

Every attack starts with reconnaissance. Before a single packet is sent, the attacker knows your technology stack, your employees, your vendors, your office layout, and your security gaps. This information is public, legal, and free. The only defense is knowing what you leak — and stopping the leak.

Imagine you're planning a heist. You don't walk into the bank blind. You watch the guards, count the cameras, learn the schedule, find the weak door. That's reconnaissance. In cyberspace, the "bank" is a company, the "guards" are firewalls, and the "weak door" is a forgotten server or a chatty employee. This module teaches you to see what everyone else misses.

SECTION 01

Passive Reconnaissance

Passive reconnaissance means gathering information without touching the target. No scans, no probes, no direct interaction. Just observation.

// WHOIS lookup — domain registration details $ whois rainfantry.github.io Domain Name: GITHUB.IO Registry Domain ID: D503300000040403267-LRMS Registrar WHOIS Server: whois.namecheap.com Registrar URL: https://namecheap.com Updated Date: 2024-01-15T08:23:17Z Creation Date: 2013-02-11T19:53:09Z Registry Expiry Date: 2025-02-11T19:53:09Z Registrar: NameCheap, Inc. Name Server: DNS1.P08.NSONE.NET Name Server: DNS2.P08.NSONE.NET DNSSEC: unsigned // What we learned: DNS provider (NSOne), registrar (NameCheap), // creation date (2013), expiry (2025), no DNSSEC (vulnerable to spoofing)
Why this matters

WHOIS tells you who manages the infrastructure. If the DNS provider is NSOne, you know the target uses modern cloud DNS. If DNSSEC is unsigned, you know the domain can be spoofed. If the registrar is NameCheap, you know the target is cost-conscious (not enterprise-grade). Every detail is a decision point.

SECTION 02

Active Reconnaissance

Active reconnaissance means touching the target. Port scans, service probes, banner grabs. This is where you get caught if you're not careful.

// Nmap port scan — find open ports $ nmap -sS -p- 192.168.1.42 Starting Nmap 7.94 ( https://nmap.org ) Nmap scan report for 192.168.1.42 Host is up (0.00032s latency). Not shown: 65530 closed ports PORT STATE SERVICE 22/tcp open ssh 445/tcp open microsoft-ds 3389/tcp open ms-wbt-server 5985/tcp open wsman 5900/tcp open vnc // SSH (22) = remote admin // SMB (445) = file sharing, potential exploit // RDP (3389) = remote desktop, credential target // WinRM (5985) = PowerShell remoting, lateral movement // VNC (5900) = remote desktop, often weak passwords

⚠ Legal warning

Active reconnaissance without authorization is illegal in most jurisdictions. The scans shown here were performed on authorized lab machines (.92, .42, .145). Never scan systems you don't own or have explicit written permission to test.

SECTION 03

OSINT — Open Source Intelligence

OSINT is reconnaissance using public sources. Social media, job postings, press releases, GitHub repositories, certificate transparency logs.

// GitHub reconnaissance — find code, commits, contributors $ curl -s https://api.github.com/users/rainfantry | jq '{login, name, company, blog, location, email, public_repos, created_at}' { "login": "rainfantry", "name": "George Wu", "company": "22nd Survey Division", "blog": "https://rainfantry.github.io/22nd-survey-division/", "location": "Sydney, Australia", "email": null, "public_repos": 92, "created_at": "2020-03-15T08:23:17Z" } // 92 public repositories — massive attack surface // Location: Sydney, Australia — timezone for timing attacks // Company: 22nd Survey Division — business context for phishing
# Enumerate all repositories for page in 1 2 3 4 5; do curl -s "https://api.github.com/users/rainfantry/repos?page=$page&per_page=100" | \ jq -r '.[] | "\(.name) | \(.language) | \(.updated_at) | \(.stargazers_count)"' done | head -20 # Output: # 22nd-survey-division | HTML | 2026-06-29T08:44:03Z | 12 # vader-rootkit | C | 2026-06-28T15:23:11Z | 8 # iron-sun | C | 2026-06-27T09:45:22Z | 5 # winrecon | Python | 2026-06-26T11:34:56Z | 3 # ghost-encoder | Python | 2026-06-25T14:12:33Z | 2 # ...
Why this is dangerous

Every repository is a potential vulnerability. The C repositories (vader-rootkit, iron-sun) suggest Windows exploitation skills. The Python repositories (winrecon, ghost-encoder) suggest automation and data exfiltration. The HTML repository (22nd-survey-division) is the public face — potential XSS or injection targets. The update timestamps show active development — the target is skilled and current.

SECTION 04

Lab Exercise: Recon Your Target

Choose a domain you own or have permission to test. Perform full reconnaissance and document findings.

  1. WHOIS lookup — document registrar, DNS, dates
  2. DNS enumeration — find A, MX, TXT, NS records
  3. Subdomain brute-force — use common names list
  4. Port scan — nmap -sS -sV -O target
  5. OSINT — GitHub, LinkedIn, social media
  6. Certificate transparency — crt.sh for subdomains
  7. Document everything in a structured report

KEY TAKEAWAYS

What You Learned

  • Reconnaissance is 80% of the attack: The more you know, the less noise you make
  • Passive recon is invisible: WHOIS, DNS, OSINT — no logs, no alerts
  • Active recon is noisy: Port scans trigger IDS/IPS — use slowly, spoof sources
  • OSINT is the goldmine: Employees leak more than servers do
  • Every detail matters: DNS provider, OS version, repository language — all inform the attack
