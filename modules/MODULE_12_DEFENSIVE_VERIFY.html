Module 12: Defensive Verification LIVE TESTED

Module 12 of 22 — Trust but verify

🧠 The Core Truth

A scan result is just text until you prove it means something. "0 detections" could mean clean, or it could mean the scanner never ran, or the file was excluded, or the definitions were stale. Verification is the discipline of proving your tools work before you bet your operation on them.

🎯 Soldier Translation

You wouldn't field a rifle without test-firing it. Same for malware. Before you deploy a tool on a target, you scan it. But how do you know the scan was real? This module teaches you to read scan results like a forensic analyst — not just the headline, but the metadata that proves the scan actually happened.

The Method: How to Verify a Scan

Step 1: Open Your Terminal

You need a command prompt. Not PowerShell. Not Developer Command Prompt. The standard Windows Command Prompt.

Press Win + R, type "cmd", press Enter
Why cmd and not PowerShell?

Kaspersky's command-line scanner (avp.exe) is designed for cmd.exe. PowerShell adds extra parsing that can break arguments. When you're verifying, you want the simplest path — no variables, no pipelines, no surprises.

Step 2: Check the File Exists

Before scanning, confirm the file is where you think it is.

dir C:\Users\gwu07\phantom_rpc.exe
Volume in drive C is Windows Volume Serial Number is XXXX-XXXX Directory of C:\Users\gwu07 06/29/2026 12:16 AM 153,600 phantom_rpc.exe 1 File(s) 153,600 bytes
What am I looking for?

Size: 153,600 bytes. If this is 0 bytes, the file is empty. If it's missing, wrong path. Date: Should match when you compiled it. If the date is old and you just built it, you're looking at the wrong file.

Step 3: Run the Kaspersky Scan

Now scan the file. This is the exact command with every flag explained.

"C:\Program Files (x86)\Kaspersky Lab\Kaspersky 21.25\avp.exe" SCAN C:\Users\gwu07\phantom_rpc.exe /i0
What does each part mean?
  • "C:\Program Files (x86)\Kaspersky Lab\Kaspersky 21.25\avp.exe" — The scanner executable. Quotes needed because of spaces in path.
  • SCAN — The command. Tells avp.exe to scan, not update or report.
  • C:\Users\gwu07\phantom_rpc.exe — The target file. Full path, no quotes needed (no spaces).
  • /i0 — Action on detect: 0 = "Ask after scan". Doesn't delete or quarantine. Lets you see the result.

Step 4: Read the Output Like a Forensic Analyst

This is the real scan output from .92 on 2026-06-29. Every line matters.

AV bases release date: 2026-06-28 18:11:00 (full) ; --- Settings --- ; Action on detect: Ask after scan ; Scan objects: All objects ; Use iChecker: Yes ; Use iSwift: Yes ; Try disinfect: Yes ; Try delete: Yes ; Try delete container: No ; Scan archives: No ; Exclude by mask: No ; Include by mask: No ; Objects to scan: ; "C:\Users\gwu07\phantom_rpc.exe" Enable = Yes Recursive = No ; ------------------ 2026-06-29 07:44:54 Scan_Objects$2997 starting 1 2026-06-29 07:44:54 C:\Users\gwu07\phantom_rpc.exe//# ok Progress 1%... 2026-06-29 07:44:54 C:\Users\gwu07\phantom_rpc.exe ok Progress 1%... 2026-06-29 07:44:56 Scan_Objects$2997 completed Info: task 'ods' finished, last error code 0 ; --- Statistics --- ; Time Start: 2026-06-29 07:44:54 ; Time Finish: 2026-06-29 07:44:56 ; Processed objects: 2 ; Total OK: 2 ; Total detected: 0 ; Suspicions: 0 ; Total skipped: 0 ; Password protected: 0 ; Corrupted: 0 ; Errors: 0 ; ------------------
How do I know this scan was REAL?

Check these 6 verification points:

  1. AV bases date: "2026-06-28 18:11:00" — Current definitions, not stale. If this says 2025 or "unknown", the scan is worthless.
  2. iChecker + iSwift: "Yes" — Both enabled. This means KAV used its full scanning engine, not a cached shortcut. If either says "No", the scan might have skipped parts.
  3. Processed objects: "2" — The file was actually read and analyzed. If this is 0, the scanner never touched it.
  4. Task completed: "task 'ods' finished, last error code 0" — No crash, no abort. Clean completion.
  5. Time taken: 2 seconds for 153KB — Reasonable. If it's 0.1 seconds, suspicious (maybe cached). If it's 60 seconds, maybe hung.
  6. Total detected: 0, Suspicions: 0: The headline. But only valid if the above 5 checks pass.
⚠️ What would a FAKE or BROKEN scan look like?
  • "Total skipped: 1" — File was excluded or unreadable. Scan didn't happen.
  • "Errors: 1" — Scanner crashed or file was locked. Result invalid.
  • "AV bases: unknown" — Definitions not loaded. Blind scan.
  • "Total OK: 0" — File wasn't processed. Maybe wrong path.
  • No output at all — avp.exe not found, wrong path, or no permission.

Step 5: Verify the File State After Scan

A real scan with "0 detections" should leave the file untouched. If the file is gone, the scan lied or the AV auto-deleted.

dir C:\Users\gwu07\phantom_rpc.exe
06/29/2026 12:16 AM 153,600 phantom_rpc.exe
What am I checking?

File still exists: If KAV detected something, it might quarantine or delete. File present = no action taken. Size unchanged: 153,600 bytes. If size changed, KAV modified it. Date unchanged: Still 12:16 AM. If date changed, KAV touched it.

Step 6: Cross-Machine Verification

One scan on one machine proves nothing. The same file scanned on a different machine with the same AV — that's verification.

# On .92 (your machine): "C:\Program Files (x86)\Kaspersky Lab\Kaspersky 21.25\avp.exe" SCAN C:\Users\gwu07\phantom_rpc.exe /i0 # On .42 (WUPC, via SSH): ssh SWu@192.168.1.42 "C:\Program Files (x86)\Kaspersky Lab\Kaspersky 21.25\avp.exe" SCAN C:\Users\swu\phantom_rpc.exe /i0
=== .92 RESULT === Total detected: 0 Suspicions: 0 Total OK: 2 === .42 RESULT === Total detected: 0 Suspicions: 0 Total OK: 2 === VERDICT === Same AV version (21.25), same definitions (2026-06-28), same result (0 detections). Cross-machine verification: PASSED.
Why cross-machine matters

A single machine could have a misconfigured exclusion list. A single scan could hit a cached result. Two independent machines with the same AV version and definitions — if both say clean, the file is clean. This is the scientific method: reproducible results.

Key Takeaways

  • A scan is only as good as its metadata: Bases date, iChecker/iSwift, processed objects, completion status — all matter
  • Verify file state post-scan: Present, unchanged size, unchanged date = no action taken
  • Cross-machine verification: Same AV, same defs, same result = reproducible science
  • Document unreachable targets: "Down" is valid data. Don't fake it.
  • Assume the layman knows nothing: Every command, every flag, every output line needs explanation
