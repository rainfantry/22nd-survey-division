Term Glossary & Crash Courses

Every neon term in the course links here. 2-minute crash courses. No prerequisites. No assumptions.

WINDOWS

Token

A data structure that represents your identity and permissions on a Windows system. Every process has a token that tells the OS "who you are" and "what you can do."

💡 Analogy: A token is like your employee ID badge. It has your photo (identity), your department (group membership), and your access level (privileges). Show the badge to the scanner, the door opens. Show a SYSTEM token to Windows, every door opens.
In the course: Module 08 (PrivEsc) — "Steal a SYSTEM token to become SYSTEM." Module 09 (Malware) — "Your token determines what the malware can do."
WINDOWS

SeImpersonatePrivilege

A Windows privilege that allows a process to impersonate another user. If you have this, you can become anyone — including SYSTEM — by intercepting their authentication.

💡 Analogy: SeImpersonatePrivilege is like having a master key that can become any other key. The janitor has it to open all offices. Service accounts have it to act on behalf of users. Attackers want it to become admin.
In the course: Module 08 — "Check whoami /priv. See SeImpersonatePrivilege? You're one exploit away from SYSTEM."
WINDOWS

UAC (User Account Control)

The "Are you sure?" dialog in Windows. When you try to do something admin-level, UAC pops up asking for confirmation. It's a security boundary — not a security barrier. It can be bypassed.

💡 Analogy: UAC is like a speed bump, not a wall. It slows you down but doesn't stop a determined driver. A real wall would require a password every time. UAC just asks "are you sure?" and most users click "yes" without reading.
In the course: Module 08 — "UAC bypass: fodhelper.exe, computerdefaults.exe, sdclt.exe. All signed Microsoft binaries that auto-elevate."
MALWARE

Dropper

A program that installs malware. It doesn't do the damage itself — it delivers the payload. Like a delivery truck that drops off a bomb, then drives away.

💡 Analogy: A dropper is like a UPS driver. The driver (dropper) isn't the thief. They just deliver the package (payload) to your door. The package contains the actual malware. The driver leaves no fingerprints.
In the course: Module 09 — "Build a dropper: download payload, write to disk, execute, self-delete."
MALWARE

IAT (Import Address Table)

A list in every Windows program that says "I need these functions from these DLLs." AV scanners read this list to detect suspicious imports (like "CreateRemoteThread" or "VirtualAllocEx").

💡 Analogy: IAT is like a shopping list. The program says "I need eggs from kernel32.dll, milk from ntdll.dll, and sugar from user32.dll." AV reads the list: "Wait, why does a calculator need lockpicks from advapi32.dll? Suspicious."
In the course: Module 09 — "Dynamic API resolution: resolve imports at runtime, not in IAT. AV sees an empty shopping list — nothing suspicious."
NETWORK

C2 (Command and Control)

The communication channel between attacker and victim. The attacker sends commands; the victim sends results. Like a radio link between headquarters and a spy in the field.

💡 Analogy: C2 is like a two-way radio. The attacker (HQ) says "take photos of the documents." The victim (spy) radios back "photos sent." If the radio is discovered, the spy is burned. So spies use disguised radios (DNS, HTTPS, Discord).
In the course: Module 16 — "Build a C2: HTTP listener, beacon interval, jitter, encrypted commands."
SOCIAL ENGINEERING

Pretext

The fictional scenario you create to manipulate someone. "I'm IT support." "I'm the new guy." "I'm your boss's assistant." The pretext gives you a reason to ask for what you want.

💡 Analogy: Pretext is like a costume. You dress as a plumber to enter the building. You dress as IT support to get passwords. The costume doesn't change who you are — it changes what people believe you are.
In the course: Module 17 — "Build a pretext: urgency + authority + fear + helpfulness. The four horsemen of social engineering."
SOCIAL ENGINEERING

MOTW (Mark of the Web)

A hidden flag Windows adds to files downloaded from the internet. It triggers warnings in Office, Edge, and Explorer. "This file came from the internet — are you sure?"

💡 Analogy: MOTW is like a customs stamp on your passport. It says "this person came from outside." The stamp triggers extra screening. But if you enter through a trusted channel (diplomatic pouch = .iso file), no stamp, no screening.
In the course: Module 17 — "MOTW bypass: deliver payload inside .iso. Windows mounts it as a local drive. No MOTW. No warning."
CRYPTO

XOR Obfuscation

A simple encryption method where each byte is combined with a key using XOR. Same operation encrypts and decrypts. Fast, reversible, breaks string signatures in AV.

💡 Analogy: XOR is like a secret language where you replace every letter with the letter +3 positions ahead. A→D, B→E. The same rule decrypts: D→A, E→B. Simple, fast, but if the enemy figures out the +3 rule, it's broken.
In the course: Module 09 — "XOR strings at compile time: 'cmd.exe' becomes garbage bytes. AV string scan sees nothing. Runtime: XOR with key 0xB5, restore 'cmd.exe', execute."
