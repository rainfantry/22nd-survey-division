// OPERATING PRINCIPLES
DOCTRINE // HOW WE OPERATE
01
Own Hardware Only
All offensive research conducted exclusively on own machines or explicitly authorised targets. No mass-targeting. No unauthorised access. No exceptions.
02
First Principles
We don't copy-paste PoCs. Every tool is built from scratch with full understanding of underlying mechanics. If we can't explain it line by line, we don't ship it.
03
Responsible Disclosure
Novel vulnerabilities go to MSRC or MITRE before anywhere else. We give vendors 90 days. We don't burn zero-days for clout.
04
Crash → Leak → Execute
The research methodology. Find the crash surface, build a leak primitive, develop controlled execution. Document every step. No magic.
05
Hunt, Don't Defend
Think like an attacker to build better defences. Understand the kill chain from ingress to persistence. See paths, see blocks, find substitutes.
06
Document Everything
No undocumented capability. Field manuals, annotated source, lesson materials. If it's not written down, it doesn't exist and can't be defended.
